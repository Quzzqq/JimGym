# Промпт для разработки Web-приложения JimGym

## Общее описание проекта

Разработай полноценное Web-приложение сайта спортзала **JimGym** на стеке:
- **React 18+** (с Create React App или Vite)
- **TypeScript**
- **Tailwind CSS**
- **React Router** для навигации

Приложение будет опубликовано на GitHub Pages. Это полностью фронтенд-проект: **никакого бэкенда, API или базы данных** — все данные захардкожены и служат только для демонстрации интерфейса.

---

## Дизайн и стиль

- **Цветовая схема:** фиолетовый и белый
  - Основной акцент: фиолетовый (#7C3AED, #6D28D9 или аналогичные оттенки)
  - Фон: белый (#FFFFFF), светло-серый (#F9FAFB) для секций
  - Текст: тёмно-серый/чёрный на светлом фоне, белый на фиолетовом
- **Стиль:** современный, лаконичный, спортивный
- **Шрифты:** используй Google Fonts — например, Inter, Poppins или Nunito
- **Иконки:** можно использовать Lucide React, Heroicons или React Icons

---

## Адаптивность и мобильная версия

**Обязательно:** приложение должно быть спроектировано mobile-first и отлично выглядеть на:
- смартфонах (320px–480px)
- планшетах (768px+)
- десктопах (1024px+)

Кнопки, навигация, формы и карточки должны быть удобными для тач-управления (достаточно большие области клика). Никаких горизонтальных скроллов на мобильных.

## Экран авторизации (Login)

### Функциональность
- Форма входа: поле **логина** (или email) и **пароля**
- Два захардкоженных аккаунта (логин + пароль на твой выбор, например один и тот же пароль для обоих):
  - `arinaUsikova` / `dashaShulyakovskaya` — второй логин
  - Пароль можно сделать одинаковым, например `gym123` или `jimgym`
- При вводе корректной пары логин-пароль — редирект на главную страницу после входа
- При неверных данных — показать сообщение об ошибке (красный текст или alert)
- Состояние "залогинен" можно хранить в `localStorage` или React state (без бэкенда это нормально)

### Текстовое примечание
Под или над формой входа должно быть заметное примечание:

> **Внимание!** Войти могут только те, кто лично получил свой аккаунт в спортзале.

Оформи его аккуратно: иконка, рамка или выделение цветом.

## Структура приложения после входа

После успешного входа пользователь попадает в основное приложение с навигацией. Нужны следующие страницы:

### 1. Главная (Home / Объявления)
- Заголовок вроде "Объявления" или "Новости зала"
- Список 3–5 захардкоженных объявлений в виде карточек, например:
  - "Обновление расписания групповых занятий с 15 марта"
  - "Технический перерыв в душевой 12–13 марта"
  - "Акция: приведёшь друга — получи скидку 10%"
  - "Поздравляем с 8 Марта! Скидка для женщин 15%"
- У каждого объявления: заголовок, короткий текст, дата (захардкожена)

### 2. Посещаемость (Attendance)
- Отображать данные посещаемости в зависимости от залогиненного пользователя
- Захардкодить данные для обоих аккаунтов:

**Для arinaUsikova:**
- Январь: 12 посещений  
- Февраль: 15 посещений  
- Март: 4 посещений (текущий месяц)

**Для dashaShulyakovskaya:**
- Январь: 10 посещений  
- Февраль: 18 посещений  
- Март: 3 посещений  

Можно показать это в виде таблицы по месяцам, списка или простых карточек. Добавь общую статистику (например, "Всего посещений: 35" за 2025 год).

### 3. Настройки (Settings)
- Отображение имени текущего пользователя (логина)
- Кнопка **Выйти** — очищает сессию и возвращает на экран авторизации

### 4. Дополнительные страницы (по желанию)
- **Расписание** — захардкоженное расписание групповых занятий по дням недели
- **Контакты** — адрес зала, телефон, часы работы (всё захардкожено)
- **Абонемент** — тип абонемента пользователя и дата окончания (разные для arinaUsikova и dashaShulyakovskaya)

## Навигация

- Нижняя навигация (bottom navigation bar) для мобильных — иконки + подписи: Главная, Посещаемость, Настройки
- На десктопе — верхняя навбар или боковое меню
- React Router: маршруты `/`, `/attendance`, `/settings`, `/schedule`, `/contacts` и т.д.

---

## Защита маршрутов

- Если пользователь не залогинен и пытается открыть `/`, `/attendance`, `/settings` и т.д. — редирект на экран авторизации
- При попытке открыть `/login` будучи залогиненным — редирект на главную

## Технические рекомендации

1. **Роутинг:** `react-router-dom` v6
2. **Состояние:** Context API для текущего пользователя и "авторизации"
3. **Структура папок:**
   - `src/components` — переиспользуемые компоненты
   - `src/pages` — страницы (Login, Home, Attendance, Settings)
   - `src/context` — AuthContext
   - `src/data` — захардкоженные данные (пользователи, объявления, посещаемость)
4. **Типизация:** интерфейсы для User, Announcement, Attendance
5. **Tailwind:** используй кастомные цвета в `tailwind.config.js` для фиолетовой палитры
